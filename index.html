PROYECTO

  • git status - Muestra los archivos que pudieron ser agregados,modificados o eliminados antes de ser subidos al repositorio
  • git add - Agrega archivos al escenario ("stage")
  • Comodines de git
    • add-git add -A / git add . : Agrega todos
    • git add --ignore-removal: Agrega nuevos y modificados, sin los eliminados
    • git add -u: Agrega modificados y eliminados, sin los nuevos
  • git log - Muestra el historial de cambios de nuestro repositorio
  • git commit - Consolida el archivo previamente creado y puesto en escenario.
  • git diff - Identifica todos los cambios de un archivo entre diferentes versiones.
  • git remote - Lista los remotos de un repositorio
  • git push - Sube cambios a remoto
  • git pull - Obtiene cambios y fusionarlos
  • git checkout --file - Cancela los cambios
  • git reset HEAD file - Quita archivos del stage
  • git branch - Crea una rama
  • git config --global user.name "Karen Salazar"-Configura en Git cual será el nombre de la persona que hará uso de él
  • git help - Ayuda de los comandos
  • git clone -Obtiene un copia al original
  • git mv-Confirma cualquier cambio de nombre o movimiento de un archivo
  • git fetch origin -Obtiene cambios de remoto
  • git merge- Fusiona cambios
  • git init -Inicializa un repositorio
  • git tag -Etiquetas
  • git rm- Sirve para borrar un archivo pero en este caso sirve para agregar al escenario el archivo que vayamos a borrar
